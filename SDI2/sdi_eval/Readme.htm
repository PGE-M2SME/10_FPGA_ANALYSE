Implementing the IP Module using Diamond SW


Instantiating the Core

The generated Tri-Rate SDI PHY IP core package contains Verilog and VHDL files. The Verilog HDL files include the black-box (<user_name>_bb.v) and instance (<username>_inst.v) templates that can be used to instantiate the core in a top-level design.

An RTL top-level reference source file that can be used as an instantiation template for the IP core is provided in \<project_dir>\sdi_eval\<username>\src\rtl\top\<family>. Users may also use this top-level reference as a template for the top-level for their complete design.

For VHDL entry, only a top-level file is generated which contains the core instantiation.

Hardware Evaluation

Lattice's IP hardware evaluation capability makes it possible to create IP cores that operate in hardware for a limited period of time (approximately four hours) without requiring the purchase on an IP license. The hardware evaluation capability is enabled by default. It can be disabled by right clicking on "Build Database" in the "Process for current sources" window of the Diamond Project Navigator. The setting is called "Hardware Evaluation" and the options are "Enable" or "Disable".

 

When the Hardware Evaluation feature is enabled in the design, it will generate a programming file that may be downloaded into the device. After initialization, the IP core will be operational for approximately four hours. After four hours, the device will stop working and it will be necessary to reprogram the device to re-enable the operation. This hardware evaluation capability is only enabled if the core has not been licensed. During implementation, a license check is performed. If the hardware evaluation feature is disabled and the IP license is not available, a pop-up window will be displayed indicating a license failure. Click "OK" in the window to complete the implementation without generating the bitstream. If a license is detected, no pop-up window is displayed and the generated core does not have any time-out restrictions.

 


Implementing the Core in a Top-Level Design

 

As described previously, the top-level file <username>_top.v(vhd) provided in \<project_dir>\sdi_eval\<username>\src\rtl\top\<family> can be used to implement the Tri-Rate SDI PHY IP core.

Push-button implementation of this top-level is supported via the Diamond project file<username>_top.ldf   located in \<project_dir>\sdi_eval\<username>\impl\<synplify|precision>\core_only

This design is intended to provide an indication of the device utilization associated with the core itself and is usually replaced with the user's top-level design.

To use the project file:

  • Select File->Open->Project in Diamond SW.
  • Browse to \<project_dir>\sdi_eval\<username>\impl\<synplify|precision>\core_only in the Open Project dialog box.
  • Select and open<username>_top.ldf. At this point, all of the files needed to support top-level synthesis and implementation will be imported to the project.
  • A logical preference file (lpf), <username>_top.lpf is available in the <project_dir>\sdi_eval\<username>\impl\<synplify|precision>\core_only directory.
  • Implement the complete design via the standard Diamond GUI flow, keeping all other options at their default values.

Running Functional Simulation

The functional simulation environment includes a configuration-specific behavioral model of the Tri-Rate SDI PHY IP, which is instantiated in an FPGA top level sample design. This FPGA top is instantiated in an evaluation testbench that provides appropriate stimulus for the Tri-Rate SDI PHY IP core. The testbench files can be found in \<project_dir>\sdi_eval\testbench. The FPGA top-level design used for simulation is the same as the loopback sample design that is provided for implementing the IP on the board. Refer to Loopback and Passthrough Sample Designs Users Guide under <project_dir>\sdi_eval   for details. This users guide also describes the test environment and ways to customize the stimulus for the testbench.

 

The generated IP core package includes the configuration-specific behavior model (<username>_beh.v, provided in \<project_dir>\sdi_eval\<username>\src\beh_rtl\<family>) for functional simulation. Models needed for simulation are provided in the corresponding \models folder.

 

Users may run the evaluation simulation on ModelSim SE* (Windows or Linux) by doing the following:

 

  • Open ModelSim.
  • Under the Tools tab, select TCL -> Execute Macro..
  • Select file \<project_dir>\sdi_eval\<username>\sim\modelsim\scripts\<username>_rtl_se.do
  • Select OK.

 

* Note:

The SERDES modules pre-compiled library must be installed before running the evaluation script. Users can download a precompiled library set from the Lattice web page Downloadable Software section. Install the PCSD library for MTI (ex: pcsd-mti_6.0-V1-1.zip). The library should be installed in the \cae_library\simulation\blackbox directory. This action will not be required if your Diamond software already includes the pre-complied library.

 

 

Users may run the evaluation simulation in Active-HDL(Windows Only) by doing the following:

  • Open Active-HDL.
  • Under the Tools tab, select Execute Macro...
  • Select file \<project_dir>\sdi_eval\<username>\sim\aldec\scripts\<username>_rtl.do
  • Select OK.

 

In the VHDL flow, the simulation uses all Verilog modules only. The user may use his VHDL design and instantiate the IP using the VHDL wrapper <username>_beh.vhd provided in \<project_dir>\sdi_eval\<username>\src\beh_rtl\<family> and use a simulator supporting mixed language simulation.

Core Specific Information

Support for Evaluation on Lattice ECP3 Video Protocol Board

Two top-level sample designs named loopback and passthru are provided with the IP package which can be used to test the IP on the Lattice ECP3 video protocol board (VPB). Please refer to Tri-Rate SDI PHY IP Users Guide for details. These designs are implemented in Verilog and use the default IP configuration in the IP GUI. Users may be able to use these designs with other IP configurations by appropriately modifying it.
